
# --- FILE: core\crypto\crypto_helper.md ---

สรุปสั้น ๆ ว่าเอกสารนี้อธิบายโครงสร้างและการทำงานของ CryptoHelper ซึ่งเป็นคลาสรวมฟังก์ชันเข้ารหัสและการจัดการกระเป๋าเงินในระบบ VKA โดยมีรายละเอียดครบถ้วน เช่น

- การสร้าง Seed Phrase 12 คำ สำหรับผู้ใช้ใหม่

- การแปลง Seed Phrase เป็นคู่กุญแจ Ed25519 (private/public key) พร้อม Prefix vka_

- การคำนวณค่าแฮช SHA-256 สำหรับบล็อกเพื่อร้อยเรียง Ledger ใน SQLite

นอกจากนี้ยังอธิบายการเชื่อมโยงระหว่าง Wallet Management ↔ Blockchain Logic และการใช้กลไกเข้ารหัสเพื่อเพิ่มความปลอดภัย ป้องกันการโจมตี และยืนยันความถูกต้องของธุรกรรม
------------------------------------------------------------------------
CryptoHelper

# ภาพรวม

CryptoHelper เป็นคลาสที่รวมฟังก์ชันช่วยเหลือด้านการเข้ารหัสและการจัดการกระเป๋าเงินในระบบ VKA โดยครอบคลุมทั้งการสร้าง Seed Phrase, การสร้างคู่กุญแจ Ed25519 และการคำนวณค่าแฮช SHA-256 สำหรับบล็อกเชน.
------------------------------------------------------------------------
# โครงสร้างหลัก

ใช้ ไลบรารีหลัก:

- crypto: สำหรับการคำนวณ SHA-256.
- bip39: สำหรับการสร้าง Seed Phrase 12 คำ.
- ed25519_hd_key: สำหรับการสร้างคู่กุญแจ Ed25519.
- convert: สำหรับการแปลงข้อมูล bytes เป็น hex string.
------------------------------------------------------------------------
ฟังก์ชันและหน้าที่

1. generate12WordsSeed()

- สุ่มสร้าง Mnemonic Seed Phrase จำนวน 12 คำ.
- ใช้สำหรับผู้ใช้ใหม่ในการกู้คืนกระเป๋าเงิน.

2. deriveKeyPairFromSeed(String seedPhrase)
- แปลง Seed Phrase ให้กลายเป็นคู่กุญแจ { privateKey, publicKey }.
- ขั้นตอน:
    - แปลง Seed Phrase เป็น Seed Bytes.
    - สร้าง Master Key จาก Seed Bytes.
    - ดึง Private Key (32 bytes) และแปลงเป็น Hex String.
    - สร้าง Public Key จาก Private Key และแปลงเป็น Hex String.
    - เพิ่ม Prefix vka_ ให้กับ Public Key เพื่อใช้ในระบบ VKA.
- คืนค่าเป็น Map<String, String> ที่ประกอบด้วย privateKey และ publicKey.

3. calculateBlockHash({...})
- คำนวณค่าแฮช SHA-256 สำหรับบล็อก.
- ขั้นตอน:
    - รวมข้อมูลธุรกรรมทั้งหมดในบล็อกเป็น String เดียว.
    - แปลง String เป็น Byte Array (UTF-8).
    - รันอัลกอริทึม SHA-256.
    - คืนค่าเป็น Hex String ความยาว 64 ตัวอักษร.

- ใช้สำหรับการร้อยเรียง Ledger ใน SQLite.
------------------------------------------------------------------------
# การเชื่อมโยงของโปรแกรม

- CryptoHelper ↔ Wallet Management: ใช้สร้าง Seed Phrase และคู่กุญแจสำหรับผู้ใช้ใหม่.

- CryptoHelper ↔ Blockchain Logic: ใช้คำนวณค่าแฮชของบล็อกเพื่อยืนยันความถูกต้องและความปลอดภัย.

- Security: การใช้ Ed25519 และ SHA-256 ทำให้ระบบมีความปลอดภัยสูงและป้องกันการโจมตี.
-------------------------------------------------------------------------
# สรุป

CryptoHelper เป็นคลาสสำคัญที่รวมฟังก์ชันการเข้ารหัสและการจัดการกระเป๋าเงินในระบบ VKA โดยรองรับทั้งการสร้าง Seed Phrase, การสร้างคู่กุญแจ Ed25519 และการคำนวณค่าแฮช SHA-256 สำหรับบล็อกเชน ทำให้ระบบมีความปลอดภัยและสามารถตรวจสอบความถูกต้องของธุรกรรมได้.

DatabaseHelper

# ภาพรวม

DatabaseHelper เป็นคลาสที่ทำหน้าที่จัดการฐานข้อมูล SQLite ของระบบ VKA โดยใช้สถาปัตยกรรมแบบ Singleton เพื่อให้ทั้งแอปเข้าถึงฐานข้อมูลผ่านช่องทางเดียว ป้องกันการเปิดหลายอินสแตนซ์ที่อาจทำให้ข้อมูลขัดแย้ง.

# โครงสร้างหลัก

- Singleton Pattern: ใช้ DatabaseHelper.instance เพื่อให้แน่ใจว่ามีเพียงอินสแตนซ์เดียวของฐานข้อมูล.

- Database: ตัวแปร _database ใช้เก็บอินสแตนซ์ของ SQLite.
------------------------------------------------------------------------
# ฟังก์ชันและหน้าที่

1. Future<Database> get database

- คืนค่าอินสแตนซ์ของฐานข้อมูล.
- หาก _database ยังไม่ถูกสร้าง จะเรียก _initDB() เพื่อสร้างใหม่.

2. _initDB(String filePath)

- กำหนดตำแหน่งไฟล์ฐานข้อมูลในระบบ Android.
- เปิดฐานข้อมูลด้วยชื่อ vka_local_ledger.db.
- หากเปิดครั้งแรกจะเรียก _createDB() เพื่อสร้างตาราง.

3. _createDB(Database db, int version)

- สร้างโครงสร้างตารางข้อมูลตามพิมพ์เขียวของระบบ VKA.

- ตารางที่สร้าง:

    1.my_wallet: เก็บข้อมูลกระเป๋าเงินของผู้ใช้ (ไม่เก็บ Private Key).
    2.local_ledger: สมุดบัญชีบล็อกเชนออฟไลน์แบบ Append-Only.
    3.peer_states: เก็บสถานะยอดสุทธิของเพื่อนบ้านเพื่อรองรับระบบกู้คืน.
    4.system_configs: เก็บการตั้งค่าระบบ เช่น เวอร์ชันกฎ อายุเงิน และลายเซ็น.
    5.peer_wallets: เก็บข้อมูลกระเป๋าของเพื่อนบ้าน เช่น alias, ประเภท, สถานะชีวิต.

4. close()

- ปิดการเชื่อมต่อฐานข้อมูลเมื่อไม่ได้ใช้งาน.
-------------------------------------------------------------------------
# การเชื่อมโยงของโปรแกรม

- DatabaseHelper ↔ SQLite: ใช้สำหรับเปิด, สร้าง, และปิดฐานข้อมูล.
- DatabaseHelper ↔ ระบบ VKA: ตารางต่าง ๆ รองรับการทำงานของระบบ เช่น การจัดเก็บธุรกรรม, การตรวจสอบสถานะเพื่อนบ้าน, และการเผยแพร่กฎใหม่.
-------------------------------------------------------------------------
# สรุป

DatabaseHelper เป็นคลาสสำคัญที่ทำหน้าที่จัดการฐานข้อมูล SQLite ของระบบ VKA โดยใช้สถาปัตยกรรม Singleton เพื่อความปลอดภัยและความเสถียรของข้อมูล พร้อมโครงสร้างตารางที่รองรับการทำงานหลักของระบบ เช่น กระเป๋าเงิน, Ledger, Peer States, System Configs และ Peer Wallets.

# ภาพรวม

LedgerRepository เป็นคลาสที่ทำหน้าที่เชื่อมต่อกับฐานข้อมูล SQLite ผ่าน DatabaseHelper เพื่อจัดการข้อมูลบล็อกเชนในระบบ VKA โดยรองรับการบันทึกบล็อกใหม่, การคำนวณยอดเงินคงเหลือ, และการดึงประวัติธุรกรรมทั้งหมด.

# ฟังก์ชันและหน้าที่

1. insertNewBlock(BlockModel block)
- บันทึกบล็อกธุรกรรมใหม่ลงในตาราง local_ledger.
- ใช้โครงสร้างแบบ Append-Only เพื่อป้องกันการแก้ไขย้อนหลัง.

2. getBalanceForAddress(String walletAddress)

- คำนวณยอดเงินคงเหลือของกระเป๋าเงินที่ระบุ.
- ขั้นตอน:

    1.ดึงบล็อกที่กระเป๋าเป็นผู้รับ (เงินเข้า).
    2.ดึงบล็อกที่กระเป๋าเป็นผู้โอน (เงินออก).
    3.ใช้ GossipEngine เพื่อดึงค่ากฎอายุเงินล่าสุด.
    4.คำนวณเงินเข้าทีละบล็อก โดยหักค่าเสื่อมตามอายุของแต่ละบล็อกผ่าน DemurrageEngine.
    5.คำนวณเงินออกตามจำนวนที่โอน.
    6.ยอดสุทธิ = เงินเข้าที่หักค่าเสื่อมแล้ว - เงินออก.

- คืนค่าเป็นยอดเงินคงเหลือที่ใช้ได้จริง.

3. getAllTransactions()

- ดึงประวัติธุรกรรมทั้งหมดจากตาราง local_ledger.
- เรียงลำดับจากใหม่สุดไปเก่าสุด.
- คืนค่าเป็น List<BlockModel>.

# --- FILE: core\economics\demurrage_engine.md ---

สรุปสั้น ๆ ว่าเอกสารนี้อธิบายโครงสร้างและการทำงานของ DemurrageEngine ซึ่งเป็นคลาสที่ใช้คำนวณมูลค่าเงิน VKA ตามกฎ Fibonacci Demurrage โดยมีรายละเอียดครบถ้วน เช่น

- ฟังก์ชัน _getFibonacci(int n) สำหรับหาลำดับฟีโบนัชชีแบบ Iterative

- ฟังก์ชัน calculateCurrentValue(...) สำหรับคำนวณมูลค่าเงินตามอายุ โดยแบ่งเป็น 3 เงื่อนไขหลัก
    - ช่วงครึ่งแรกของอายุเงิน → มูลค่าเต็ม 100%
    - ครบอายุขัย → มูลค่าเป็น 0%
    - ครึ่งหลังของอายุเงิน → ลดมูลค่าแบบก้าวกระโดดตามลำดับฟีโบนัชชี

- การป้องกันค่าติดลบและการปัดเศษทศนิยมให้ตรงตามมาตรฐานหน่วยเงิน

ในเอกสารยังอธิบายการเชื่อมโยงระหว่าง DemurrageEngine ↔ LedgerRepository ที่ใช้ฟังก์ชันนี้ในการคำนวณยอดเงินคงเหลือหลังหักค่าเสื่อม ทำให้ระบบ VKA มีความโปร่งใสและยุติธรรมในการจัดการมูลค่าเงินตามเวลา
--------------------------------------------------------------------------
DemurrageEngine

# ภาพรวม

คลาส DemurrageEngine ถูกออกแบบมาเพื่อคำนวณมูลค่าเงินตามกฎ Fibonacci Demurrage ซึ่งเป็นกลไกการเสื่อมค่าของเงินตามอายุ โดยใช้หลักการฟีโบนัชชีในการเร่งการลดมูลค่าในช่วงครึ่งหลังของอายุเงิน
---------------------------------------------------------------------------
ฟังก์ชันสำคัญ

1. _getFibonacci(int n)

- หน้าที่: คำนวณหาค่าลำดับฟีโบนัชชีตัวที่ n ด้วยวิธี Iterative (วนลูป) เพื่อความเร็วสูง

- ตัวแปรสำคัญ:
    - a และ b ใช้เก็บค่าลำดับก่อนหน้าและปัจจุบัน
    - emp ใช้เก็บผลรวมชั่วคราวของ a + b

- การทำงาน:
    - ถ้า n <= 0 คืนค่า 0
    - ถ้า n == 1 คืนค่า 1
    - ถ้า n > 1 จะวนลูปบวกค่าฟีโบนัชชีไปเรื่อย ๆ จนถึงตำแหน่งที่ต้องการ

2. calculateCurrentValue({...})

- หน้าที่: คำนวณมูลค่าเงินคงเหลือปัจจุบันตามกฎ Fibonacci Demurrage

- พารามิเตอร์:

    - originalAmount → จำนวนเงินเริ่มต้น
    - txTimestamp → เวลาที่ได้รับเงิน (หน่วยวินาที)
    - lifespanDays → อายุขัยเต็มของเงิน (ค่าเริ่มต้น 30 วัน)
    - currentTimestampOverride → ใช้แทนเวลาปัจจุบันสำหรับ Unit Test

- ตัวแปรสำคัญ:
    - nowSeconds → เวลาปัจจุบันในหน่วยวินาที
    - secondsInDay → จำนวนวินาทีในหนึ่งวัน (86400)
    - ageInDays → อายุของเงินที่ผ่านไปแล้วกี่วัน
    - halfLife → ครึ่งหนึ่งของอายุขัยเงิน
    - fibIndex → ดัชนีฟีโบนัชชีที่ใช้คำนวณเปอร์เซ็นต์การเสื่อมค่า
    - penaltyPercent → ค่าฟีโบนัชชีที่ได้มาใช้เป็นเปอร์เซ็นต์การหักมูลค่า
    - remainingFactor → อัตราส่วนมูลค่าที่เหลืออยู่หลังหักเปอร์เซ็นต์
    - finalValue → มูลค่าเงินสุดท้ายหลังการคำนวณ

# ฟังก์ชันหลัก

1. startCardReadingMode({...})

- หน้าที่: เปิดโหมดอ่านบัตร NFC สำหรับกลุ่มเปราะบาง

- พารามิเตอร์:

    - onCardReadSuccess → callback เมื่ออ่านบัตรสำเร็จ
    - onError → callback เมื่อเกิดข้อผิดพลาด
    - onStatusUpdate → callback สำหรับอัปเดตสถานะ

- การทำงาน:

1.ตรวจสอบว่าอุปกรณ์รองรับ NFC และเปิดใช้งานแล้ว
2.เปิดเซสชันการค้นหาแท็ก NFC (iso14443, iso15693)
3.เมื่อพบแท็ก → อ่านข้อมูลดิบ (mock payload) และส่งกลับผ่าน onCardReadSuccess
4.ปิดเซสชันหลังอ่านเสร็จ

2. writeWalletToCard(targetAddress, {...})

- หน้าที่: เขียนข้อมูลกระเป๋าเงินลงบัตร NFC (ใช้โดยผู้ใหญ่บ้าน)

- พารามิเตอร์:

    - targetAddress → ที่อยู่กระเป๋าเงินที่จะเขียนลงบัตร
    - onWriteSuccess → callback เมื่อเขียนสำเร็จ
    - onError → callback เมื่อเกิดข้อผิดพลาด
    - onStatusUpdate → callback สำหรับอัปเดตสถานะ

- การทำงาน:

    1.ตรวจสอบว่าอุปกรณ์รองรับ NFC และเปิดใช้งานแล้ว
    2.เปิดเซสชันการค้นหาแท็ก NFC
    3.เมื่อพบแท็ก → แปลง targetAddress เป็น byte array และสร้าง payload (Uint8List)
    4.ส่งข้อมูลลงบัตร NFC และเรียก onWriteSuccess
    5.ปิดเซสชันหลังเขียนเสร็จ

3. stopNfcSession([dynamic a])

- หน้าที่: ปิดเซสชัน NFC ที่กำลังทำงานอยู่

- การทำงาน:

    - เรียก NfcManager.instance.stopSession() เพื่อหยุดการทำงาน
    - แสดงข้อความ debug ว่าเซสชันถูกปิดแล้ว

BlockModel

# ภาพรวม

BlockModel เป็นคลาสที่ใช้แทนข้อมูลของบล็อกในระบบบล็อกเชน VKA โดยเก็บรายละเอียดธุรกรรมและข้อมูลที่จำเป็นสำหรับการตรวจสอบความถูกต้อง เช่น แฮช, ลายเซ็นดิจิทัล, และดัชนีบล็อก.
------------------------------------------------------------------
# โครงสร้างหลัก

- Fields (ตัวแปรสำคัญ):

    - blockIndex: ดัชนีบล็อก (สามารถเป็น null เพื่อให้ SQLite กำหนดค่า AutoIncrement).
    - previousHash: ค่าแฮชของบล็อกก่อนหน้า.
    - senderAddress: ที่อยู่กระเป๋าของผู้โอน.
    - receiverAddress: ที่อยู่กระเป๋าของผู้รับ.
    - amount: จำนวนเงินที่โอน.
    - nonce: ลำดับการโอนของผู้โอน ใช้ป้องกัน Replay Attack.
    - timestamp: เวลาที่เกิดธุรกรรม (Unix Timestamp).
    - signature: ลายเซ็นดิจิทัลที่ผู้โอนเซ็นอนุมัติ.
    - blockHash: ค่าแฮช SHA-256 ของบล็อก.
-----------------------------------------------------------------    
# Constructor

BlockModel({
  this.blockIndex,
  required this.previousHash,
  required this.senderAddress,
  required this.receiverAddress,
  required this.amount,
  required this.nonce,
  required this.timestamp,
  required this.signature,
  required this.blockHash,
});

- ใช้สร้างอินสแตนซ์ใหม่ของบล็อก โดยต้องระบุค่าที่จำเป็นทั้งหมด ยกเว้น blockIndex ที่สามารถปล่อยว่างได้.
----------------------------------------------------------------------
# ฟังก์ชันและหน้าที่

1. toMap()

- แปลงข้อมูลในคลาสเป็น Map<String, dynamic> เพื่อเตรียมบันทึกลง SQLite.
- ใช้สำหรับการ Insert ข้อมูลธุรกรรมลงฐานข้อมูล.

2. fromMap(Map<String, dynamic> map)

- Factory constructor สำหรับสร้าง BlockModel จากข้อมูลที่อ่านมาจาก SQLite.
- แปลงค่าที่ดึงจากฐานข้อมูลกลับมาเป็นวัตถุในแอป.
- มีการจัดการกรณีที่ SQLite คืนค่า amount เป็น int หรือ double โดยบังคับให้เป็น double.
---------------------------------------------------------------------
# การเชื่อมโยงของโปรแกรม

- BlockModel ↔ SQLite: ใช้ toMap() และ fromMap() ในการบันทึกและอ่านข้อมูลธุรกรรม.
- BlockModel ↔ Blockchain Logic: เป็นโครงสร้างข้อมูลหลักที่ใช้ในการตรวจสอบความถูกต้องของธุรกรรมและการสร้างบล็อกใหม่.
- Security: ใช้ nonce, signature, และ blockHash เพื่อป้องกันการโจมตีและยืนยันความถูกต้องของธุรกรรม.
--------------------------------------------------------------------
# สรุป

BlockModel เป็นคลาสสำคัญที่ทำหน้าที่เป็นตัวแทนของบล็อกในระบบ VKA blockchain โดยมีฟังก์ชันสำหรับการบันทึกและอ่านข้อมูลจาก SQLite และมีตัวแปรที่จำเป็นสำหรับการตรวจสอบความถูกต้องและความปลอดภัยของธุรกรรม.

Future<void> refreshDashboard() async {
  _isLoading = true;
  notifyListeners();

  try {
    final db = await DatabaseHelper.instance.database;
    final List<Map<String, dynamic>> walletRows = await db.query('my_wallet', limit: 1);

    if (walletRows.isNotEmpty) {
      _myAddress = walletRows.first['wallet_address'] as String;
      _myAlias = (walletRows.first['alias_name'] as String?) ?? "ผู้ใช้งาน VKA";

      final String lifeStatus = (walletRows.first['life_status'] as String?) ?? "ALIVE";
      if (lifeStatus == "DECEASED") {
        _isDeceased = true;
        const secureStorage = FlutterSecureStorage();
        await secureStorage.deleteAll();
      }
    } else {
      _myAddress = "vka_unknown_address";
    }

    if (_isDeceased) {
      _currentBalance = 0.0;
      _transactions = [];
      return;
    }

    _currentBalance = await _ledgerRepo.getBalanceForAddress(_myAddress);
    _transactions = await _ledgerRepo.getAllTransactions();
  } catch (e) {
    debugPrint("❌ แดชบอร์ดดึงข้อมูลล้มเหลว: $e");
  } finally {
    _isLoading = false;
    notifyListeners();
  }
}

void main() async {
  WidgetsFlutterBinding.ensureInitialized();
  const secureStorage = FlutterSecureStorage();
  final String? savedPrivateKey = await secureStorage.read(key: 'vka_private_key');

RegistrationController เป็นคลาสที่สืบทอดจาก ChangeNotifier เพื่อใช้ควบคุมการทำงานของการลงทะเบียนกระเป๋าเงิน (Wallet) ทั้งในโหมดสมาร์ทโฟนและโหมดออกบัตร NFC โดยมีการเชื่อมโยงกับระบบ CryptoHelper, SecureStorageService, DatabaseHelper, และ NfcService

📦 การนำเข้า (Imports)
- package:flutter/material.dart → ใช้สำหรับ UI และการจัดการ State
- crypto_helper.dart → ใช้สำหรับสร้าง Seed และ Key Pair (Public/Private Key)
- secure_storage_service.dart → ใช้สำหรับบันทึกข้อมูลลับ เช่น PIN และ - Private Key ลงใน Secure Storage
- database_helper.dart → ใช้สำหรับบันทึกข้อมูลลง SQLite Database
- nfc_service.dart → ใช้สำหรับสื่อสารกับบัตร NFC เช่น การเขียนข้อมูลลงบัตร

🏗 Class: RegistrationController
class RegistrationController extends ChangeNotifier {
  final _secureStorage = SecureStorageService();
  final _nfcService = NfcService();

  String _generatedSeed = "";
  String get generatedSeed => _generatedSeed;

  bool _isCreated = false;
  bool get isCreated => _isCreated;

  String _nfcStatusMessage = "";
  String get nfcStatusMessage => _nfcStatusMessage;
}

🔎 หน้าที่
- จัดการการสร้าง Seed และ Key Pair สำหรับ Wallet
- บันทึกข้อมูล Wallet ลง Secure Storage และ Database
- จัดการการออกบัตร NFC และบันทึกสถานะการทำงาน
- แจ้งเตือน UI ผ่าน notifyListeners() เมื่อมีการเปลี่ยนแปลงค่า

⚙️ ฟังก์ชันสำคัญ
1. prepareNewWallet()

  _generatedSeed = CryptoHelper.generate12WordsSeed();
  notifyListeners();

- สร้าง Seed แบบ 12 คำใหม่สำหรับ Wallet
- แจ้ง UI ให้รีเฟรชเมื่อ Seed ถูกสร้าง

2. confirmAndRegisterWallet()
Future<void> confirmAndRegisterWallet({
  required String aliasName,
  required String pin,
}) async { ... }

- ช้สำหรับผู้ใช้ที่มีสมาร์ทโฟน
- สร้าง Key Pair จาก Seed
- บันทึก PIN และข้อมูลลับลง Secure Storage
- บันทึกข้อมูล Wallet ลง SQLite (my_wallet table)
- อัปเดตสถานะ _isCreated = true

3. registerAndIssueNfcCard()
Future<void> registerAndIssueNfcCard({required String holderName}) async { ... }
- ใช้สำหรับโหมดออกบัตร NFC (เช่น เด็ก/คนชรา)
- สร้าง Key Pair และนำ Public Key ไปเขียนลงบัตร NFC
- อัปเดตข้อความสถานะผ่าน _nfcStatusMessage
- หากเขียนสำเร็จ จะบันทึกข้อมูลบัตรลง SQLite (peer_states table) พร้อมทุนตั้งต้น 1000 VKA
- อัปเดตสถานะ _isCreated = true

4. cancelNfcSession()
void cancelNfcSession() {
  _nfcService.stopNfcSession();
  _nfcStatusMessage = "";
  notifyListeners();
}
- ยกเลิกการทำงานของ NFC เมื่อผู้ใช้กดยกเลิกกลางคัน
- เคลียร์ข้อความสถานะ

🔗 การเชื่อมโยงของโปรแกรม
1.CryptoHelper → สร้าง Seed และ Key Pair
2.SecureStorageService → บันทึกข้อมูลลับ (PIN, Private Key, Seed)
3.DatabaseHelper → บันทึกข้อมูล Wallet และ Peer States ลง SQLite
4.NfcService → เขียนข้อมูล Public Key ลงบัตร NFC และอัปเดตสถานะ

✅ สรุป
RegistrationController เป็นตัวกลางที่จัดการการสร้าง Wallet ทั้งในโหมดสมาร์ทโฟนและโหมดออกบัตร NFC โดยเชื่อมโยงกับระบบเข้ารหัส, การจัดเก็บข้อมูลลับ, ฐานข้อมูล และการสื่อสารกับบัตร NFC เพื่อให้การลงทะเบียนเป็นไปอย่างปลอดภัยและมีประสิทธิภาพ

1. emitNewGlobalRules({nextVersion, newLifespanDays})

- ดึง Private Key ของผู้ใหญ่บ้านจาก Secure Storage
- สร้างข้อความดิบที่รวมข้อมูลเวอร์ชัน, อายุเงินใหม่, Public Key และ Timestamp
- เซ็นชื่อด้วยอัลกอริทึม Ed25519 ผ่านแพ็กเกจ cryptography
- แปลงลายเซ็นเป็น Hex string

- สร้าง Gossip Packet ในรูปแบบ JSON:

{
  "type": "SYS_CONFIG",
  "config_version": nextVersion,
  "lifespan_days": newLifespanDays,
  "issued_by": VILLAGE_HEAD_PUB_KEY,
  "timestamp": nowTimestamp,
  "signature": signatureHex
}

- สร้าง QR Payload ที่ฝัง Gossip Packet:
    vka://invoice?to=VILLAGE_HEAD_PUB_KEY&amount=0.00&gossip_packet=JSON_STRING

- อัปเดต _gossipQrPayload และแจ้ง UI

2. resetPanel()
